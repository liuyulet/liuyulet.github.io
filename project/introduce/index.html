# 项目介绍

# 项目背景

自从接触了很多设备之后，一直想搞一个平台，能方便管理设备、方便进行设备接入、方便进行设备控制、方便进行设备数据展示。

并且云端仅仅管理设备，边缘负责接入设备。两者分开，各司其职，

所以，筹备了很久，lana总算有了一个雏形。

特点：

  • 接入协议简单...
  • 专注于云平台的设备管理...
  • 完善的情景模式（规则场景）...
  • 通俗易懂的代码与使用习惯，方便后期修改（实际上是自己懒，尽量简单实现）...
  • 为后期diy设备做个准备...

目标

  1. 完成物联网平台的整体开发，简单部署与使用。
  2. 人生很短，做一些自己喜欢做的事情吧。

# 项目技术栈

序号 项目架构 架构描述
1 后端技术栈 springboot3.3 (opens new window) + springSecurity6.2 (opens new window) + Mybatis-Plus3.5.5 (opens new window) + jwt0.11 (opens new window) + AviatorScript5.4.3 (opens new window) + mica-mqtt2.4.4 (opens new window) + caffeine3.1.8 (opens new window) + redisson3.21.3 (opens new window) + hutool5.8.21 (opens new window) + captcha1.6.2 (opens new window) + knife4j4.3 (opens new window) + mapstruct1.5.5 (opens new window) + dynamic4.2 (opens new window) ...
2 必要的服务 mysql8 + TDengine3.2 + redis7.0 + minio8.5.1 + jdk17 + node.js
3 前端技术栈 Vue3 + Element-Plus（由scui (opens new window)项目进行修改开发）

# 功能说明

# 1、基础框架选型、数据库和缓存整合

序号 功能模块 子功能描述
1 基础框架选型 springboot3.3 + SpringSecurity6.2 + Mybatis-Plus3.5.5 + jwt0.11 + java17 + minio8.5.1 + knife4j4.3 + mapstruct1.5.5 + dynamic4.2 + AviatorScript5.4.3 + mica-mqtt2.4.4
2 常用数据库 mysql8 + TDengine3.2
3 缓存 redis7.0 + caffeine3.1.8
4 基础 base 封装 安全认证封装、缓存实现与封装、文件上传下载封装、数据库mybatis封装、
接口操作日志封装、knife4j接口文档封装、全局异常封装、默认分页与返回结果封装
、默认树形节点处理、xss封装 、通用工具类处理
5 基础 system 功能 详见下方表格

# 2、基础 system 功能

序号 功能模块 子功能描述
1 字典管理 字典增删改查，封装方法
2 部门管理 部门增删改查，以及关联关系
3 菜单管理 进行增删改查，以及接口权限的设置
4 角色权限管理 进行增删改查，菜单权限、数据权限
5 用户管理 进行增删改查，进行组织与员工的联查，实现角色分配，与密码的重置
6 不同账户管理 登录注册、不同角色权限菜单调整
7 日志管理 将所有的操作日志进行记录与查询
8 代码优化 持续修改、优化代码、解决不合理的bug、调整为符合最新项目进度的结构

# 功能计划大纲1.0版本（进行中）

# 1、服务端功能进展

序号 功能模块 子功能描述 描述
1 设备信息 产品管理 产品配置，以及设置产品的具体属性和接入信息
2 设备信息 设备管理 配置具体的设备信息
3 设备信息 设备分组 将设备进行分组处理
4 接入管理 接入协议 主要是维护在系统中，设备的接入方式
5 接入管理 客户端管理(开发中) 主要是对已经连接的客户端的管理
6 规则编排 情景模式 通过图形化节点的方式维护情景模式
7 规则事件 情景日志 展示情景模式中的执行日志
8 预警信息 消息记录 符合预警消息的数据会被处理成预警消息
9 组态管理 组态设置(开发中) 将设备信息在2D大屏上进行设置
10 组态管理 组态展示(开发中) 2D大屏展示

# 2、边缘端功能进展

序号 功能模块 子功能描述
1 设备管理(开发中) 支持PLC(西门子、欧姆龙、三菱)、modbus、modbus-tcp、modbus-rtu、mqtt
、http、tcp、udp、websocket、mqtt-ws协议，并接入对应的设备
2 认证同步(开发中) 统一边缘认证，支持设备信息同步、支持设备认证、以及设备数据上报、云边联动
